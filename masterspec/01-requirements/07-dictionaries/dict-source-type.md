# Справочник: Тип источника диалога

## Назначение
Классифицирует диалоги-источники по типу — определяет правило гейтинга (личка доставляется всегда;
каналы/группы — по белому списку или упоминанию) и формат тега источника в письме.

## Источник структуры
- Сущность: -> cdm-bridge / Диалог / Источник (атрибут «тип источника»)

## Значения

| Код | Наименование | Правило доставки | Пример тега |
|-----|--------------|------------------|-------------|
| dm | Личка | Доставляется всегда (адресована по определению) | `dm:<отправитель>` |
| channel | Канал | Доставляется по белому списку или упоминанию | `ch:<канал>` |
| group | Группа | Доставляется по белому списку или упоминанию | `gr:<группа>` |
| topic | Топик группы | Доставляется по белому списку или упоминанию; батчинг — по топику | `gr:<группа>/<топик>` |

## Режим ведения
- Владелец: продукт (фиксированный перечень типов).
- Частота изменений: очень редко (стабильная предметная классификация).
- Процесс добавления значения: только при расширении охватываемых видов диалогов Telegram.

## Инварианты
- Коды уникальны; каждый отслеживаемый диалог относится ровно к одному типу источника.
- Тип `dm` не подчиняется правилам белого списка/упоминания; `channel`/`group`/`topic` — подчиняются.

## Связи
- Концептуальная модель: -> cdm-bridge
- Используется в функциях: -> fn-dm-batch-to-email, -> fn-channel-update-to-email
- Бизнес-правила: -> rules-gating, -> rules-delivery
