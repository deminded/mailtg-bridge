# Справочник: Способ представления вложения в письме

## Назначение
Определяет, как элемент медиа/вложения Telegram попадает в письмо: изображение — встроенным в тело,
малый файл — приложением, крупный — текстовым указанием. Используется при формировании письма.

## Источник структуры
- Сущность: -> cdm-bridge / Мостовое письмо (встроенные изображения / приложенные файлы / указания)

## Значения

| Код | Наименование | Условие применения | Что попадает в письмо |
|-----|--------------|--------------------|-----------------------|
| inline-image | Встроенное изображение | Вложение является изображением | Изображение в теле письма (инлайн) |
| attached-file | Приложенный файл | Не-изображение размером ≤ порога | Файл-вложение письма |
| notice-only | Текстовое указание | Не-изображение размером > порога | Строка «имя / размер / тип», без тела |
| weblink | Ссылка на веб-диск | Крупное вложение (задел версии 2) | Ссылка на скачивание (не реализуется в версии 1) |

## Режим ведения
- Владелец: продукт; порог размера — параметр конфигурации.
- Частота изменений: редко (значение `weblink` активируется в версии 2).
- Процесс добавления значения: при расширении способов доставки вложений.

## Инварианты
- Коды уникальны; каждому вложению письма сопоставлен ровно один способ представления.
- Изображение всегда получает `inline-image`, независимо от порога размера не-изображений.
- Значение `weblink` в версии 1 не применяется (только задел).

## Связи
- Концептуальная модель: -> cdm-bridge
- Используется в функциях: -> fn-media-in-email
- Бизнес-правила: -> rules-delivery
