# Приёмочный тест: управление мостом письмом (вкл/выкл)

## Проверяет
- Функция: -> fn-bridge-control-by-email
- AC: AC-01, AC-02, AC-03
<!-- Fold guardrail exceptions-first: недоверенный отправитель — named observable outcome (игнор + лог). -->

## Предусловия
В конфигурации задан адрес пользователя U (доверенный отправитель) и (опционально) секрет-токен темы;
команды принимаются на ящике моста B; мост включён.

## Входные данные
- Письмо-команда «выключить» с адреса пользователя U (на ящик моста B).
- Письмо-команда «включить» с адреса пользователя U (на ящик моста B).
- Письмо-команда «выключить» с постороннего адреса X (не U).

## Шаги
1. Отправить команду «выключить» с U; дождаться обработки. Затем прислать новое личное сообщение.
2. Отправить команду «включить» с U; дождаться обработки. Затем прислать новое личное сообщение.
3. Отправить команду «выключить» с постороннего адреса X; дождаться обработки.

## Ожидаемый результат
- После «выключить» с U мост не доставляет новых писем и не публикует ответы; приходит подтверждение.
- После «включить» с U доставка возобновляется; приходит подтверждение; новое сообщение доставлено.
- Команда с X (≠ U) игнорируется: состояние не меняется, факт фиксируется в логе, подтверждение не шлётся.

## Постусловия
Состояние моста соответствует последней доверенной команде и переживает перезапуск.

- Функция: -> fn-bridge-control-by-email
- Концептуальная модель: -> cdm-bridge
