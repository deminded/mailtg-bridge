# Приёмочный тест: ответ из почты в Telegram от лица пользователя

## Проверяет
- Функция: -> fn-email-reply-to-tg
- AC: AC-01, AC-02, AC-03, AC-04

## Предусловия
Существует доставленное bridged-письмо диалога D с записью связки (отправлено с ящика моста B на
адрес пользователя U); мост включён.

## Входные данные
- Ответ письмом с адреса пользователя U на bridged-письмо диалога D (приходит на ящик моста B).
- Ответ письмом на произвольное не-bridged письмо (без записи связки).
- Ответ на bridged-письмо диалога D с постороннего адреса X (не U).

## Шаги
1. Ответить с адреса U на bridged-письмо диалога D; дождаться такта опроса ящика моста B.
2. Ответить на не-bridged письмо; дождаться такта опроса.
3. Ответить с постороннего адреса X на bridged-письмо диалога D; дождаться такта опроса.

## Ожидаемый результат
- Текст первого ответа появляется в диалоге D в Telegram от лица пользователя.
- Второй ответ игнорируется: в Telegram ничего не публикуется.
- Ответ маршрутизирован по записи связки в диалог D (по диалогу, не по отдельному сообщению).
- Третий ответ (с адреса X ≠ U) игнорируется: в Telegram ничего не публикуется, факт логируется.

## Постусловия
Ответ доставлен в исходный диалог; ответ без записи связки не порождает публикации.

## Связи
- Функция: -> fn-email-reply-to-tg
- Концептуальная модель: -> cdm-bridge
