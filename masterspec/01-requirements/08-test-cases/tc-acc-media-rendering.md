# Приёмочный тест: представление медиа и вложений в письме

## Проверяет
- Функция: -> fn-media-in-email
- AC: AC-01, AC-02, AC-03, AC-04

## Предусловия
Порог размера вложений задан в конфигурации значением P.

## Входные данные
- Сообщение с изображением.
- Сообщение с не-изображением размером ≤ P.
- Сообщение с не-изображением размером > P.

## Шаги
1. Прислать сообщение с изображением; дождаться письма.
2. Прислать сообщение с малым файлом (≤ P); дождаться письма.
3. Прислать сообщение с крупным файлом (> P); дождаться письма.
4. Изменить порог P в конфигурации; повторить шаг 2 с файлом, ставшим крупным.

## Ожидаемый результат
- Изображение отображается встроенным в теле письма (не приложением).
- Малый файл приходит приложенным файлом.
- Крупный файл приходит текстовым указанием (имя/размер/тип) без тела.
- После смены порога граница файл/указание меняется без изменения кода.

## Постусловия
Тело письма — HTML со встроенными изображениями и текстовым запасным вариантом.

## Связи
- Функция: -> fn-media-in-email
- Концептуальная модель: -> cdm-bridge
- Справочники: -> dict-media-disposition
