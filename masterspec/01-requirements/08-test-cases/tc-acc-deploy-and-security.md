## Предусловия
Чистая машина; доступ к аккаунту Telegram (телефон для кода); открытый репозиторий поставки.

## Входные данные
- Инструкция поставки (README), пример конфигурации.
- Секрет-скан-инструмент для проверки репозитория.

## Шаги
1. Развернуть мост на чистой машине по README, засекая время до первого доставленного письма.
2. Выполнить интерактивный setup (телефон/код/2FA), затем перезапустить как фоновый сервис.
3. Проверить права файла сессии и файлов секретов.
4. Прогнать секрет-скан по репозиторию; проверить, что сессия/секреты вне репозитория.
5. Инспектировать логи после нескольких тактов доставки.
6. Сменить почтового провайдера правкой конфигурации, не трогая код.
7. Открыть README и найти раздел рисков userbot.
8. По README завести ящик моста B (выделенный аккаунт с app-паролем); задать креды B (IMAP/SMTP) и
   адрес пользователя U в конфигурации.

## Ожидаемый результат
- Развёртывание до первого письма укладывается в ≤ 10 минут.
- После setup создан файл сессии; фоновый сервис стартует неинтерактивно; при недействительной
  сессии сервис явно требует повторной инициализации.
- Права файла сессии и файлов секретов = 600.
- Секрет-скан репозитория чист (0 находок); сессия и секреты вне репозитория.
- Логи не содержат тел сообщений и секретов.
- Смена провайдера достигнута правкой конфигурации без изменения кода (0 изменённых файлов кода).
- README содержит явный раздел о риске userbot и цене компрометации сессии.
- README содержит раздел о заведении ящика моста B (выделенный аккаунт, app-пароль); пример
  конфигурации покрывает креды ящика B (IMAP/SMTP) и адрес пользователя U (100% обязательных параметров).

## Постусловия
Развёрнутый экземпляр работает; секреты и сессия защищены правами доступа и размещением вне
репозитория; провайдер сменяем конфигурацией.
