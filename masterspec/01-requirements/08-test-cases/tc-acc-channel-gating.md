# Приёмочный тест: гейтинг каналов и групп (белый список/упоминание)

## Проверяет
- Функция: -> fn-channel-update-to-email
- AC: AC-01, AC-02, AC-03, AC-04, AC-05, AC-06

## Предусловия
В конфигурации: канал W — в белом списке; канал X — не в белом списке; политика упоминаний
включена.

## Входные данные
- Сообщение в канале W (вайтлист).
- Сообщение с прямым упоминанием пользователя в канале X (невайтлист).
- Обычное сообщение в канале X без упоминания.

## Шаги
1. Опубликовать сообщение в W; дождаться такта.
2. Опубликовать в X сообщение с упоминанием пользователя; дождаться такта.
3. Опубликовать в X сообщение без упоминания; дождаться такта.
4. Убрать W из белого списка правкой конфигурации; опубликовать в W; дождаться такта.
5. Кликнуть глубокую ссылку доставленного сообщения из W (публичный) и из приватной группы/топика;
   сверить представление письма с видом в клиенте.

## Ожидаемый результат
- Сообщение из W доставлено письмом с тегом источника.
- Сообщение с упоминанием из X доставлено письмом.
- Сообщение из X без упоминания не доставлено (письма нет).
- После правки конфигурации W перестаёт доставляться — поведение изменилось без изменения кода.
- Глубокая ссылка открывает именно это сообщение (публичные — `t.me/<user|channel>/<msg>`, приватные
  — `t.me/c/<chat>/<msg>`, топик — в свой топик).
- Автор, локализованное время, форматирование и цитата-ответ видны как в клиенте без потери
  форматирования.

## Постусловия
Доставлены только адресованные обновления; смена политики достигнута только конфигурацией.
