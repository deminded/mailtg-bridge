# Правила: гейтинг адресованности (что доставляется)

## Бизнес-правила
- Все личные сообщения (личка) доставляются всегда — без дополнительной проверки адресованности.
- Обновление канала или группы доставляется ТОЛЬКО если источник входит в белый список ИЛИ
  содержит прямое упоминание пользователя.
- Обновление из невайтлист-источника без упоминания по умолчанию НЕ доставляется.
- Смена политики доставки выполняется только правкой конфигурации, без изменения кода.

## Инженерные правила
- Политика упоминаний конфигурируема в трёх режимах: все источники / выбранные источники /
  форсирование упоминаний по списку каналов (да/нет/выборочно).
- Гейт применяется к каждому сообщению до включения в батч; решение гейта — «пропустить» или
  «отсеять», без промежуточных состояний.
- Контентная фильтрация (по смыслу текста) в гейт не входит — только правила адресованности.

## Внешние ограничения (если есть)
- Нет внешних (регуляторных) ограничений; политика — установка продукта/владельца.

## Принципы принятия решений
- При сомнении для личка — доставлять (личка приоритетна); для каналов/групп — не доставлять без
  явного основания (белый список или упоминание), чтобы не воспроизводить шум.

## Связи
