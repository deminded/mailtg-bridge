# Функция: Представление медиа и вложений Telegram в письме

## Цель
Показать содержимое медиа из Telegram в письме удобно и без раздувания ящика: изображения — сразу
видны в теле, прочие вложения — по размеру.

## Инициатор
Система (в ходе формирования письма-батча).

## Триггер
Наличие у сообщения батча вложений (изображений и/или прочих файлов).

## Предусловия
Формируется письмо-батч диалога; задан порог размера вложений в конфигурации.

## Основной поток
1. Для каждого изображения сообщения система встраивает его в тело письма (инлайн, в HTML-теле).
2. Для каждого не-изображения размером не выше порога система прикладывает его файлом.
3. Для каждого не-изображения размером выше порога система добавляет текстовое указание
   (имя / размер / тип) без тела вложения.
4. Система собирает тело письма как HTML со встроенными изображениями и текстовым запасным вариантом.

## Альтернативные потоки
- Сообщение без вложений → тело содержит только текст (с запасным текстовым вариантом).
- Несколько изображений в сообщении → все встраиваются инлайн, порядок сохраняется.
- (Задел версии 2) крупное вложение → ссылка на веб-диск вместо текстового указания.

## Исключительные потоки (ошибки, отказы)
- Скачивание вложения из Telegram не удалось → в письмо помещается указание о недоступном вложении;
  наблюдаемый исход: письмо всё равно доставлено, текст не блокируется, вложение помечено
  недоступным.
- Изображение не удалось встроить → оно понижается до текстового указания (имя/тип), доставка тела
  не срывается.

## Результат
Письмо, где изображения видны в теле, малые вложения приложены файлами, крупные — обозначены
текстовым указанием.

## Постусловия
Тело письма — валидный HTML с текстовым запасным вариантом; размер письма ограничен порогом
вложений.

## Критерии приёмки
- AC-01: дано сообщение с изображением, когда формируется письмо, тогда изображение отображается
  встроенным в теле письма (не приложением).
- AC-02: дано не-изображение размером ≤ порога, когда формируется письмо, тогда оно приходит
  приложенным файлом.
- AC-03: дано не-изображение размером > порога, когда формируется письмо, тогда оно приходит
  текстовым указанием (имя/размер/тип) без тела.
- AC-04: дано изменение порога в конфигурации, когда формируется письмо, тогда граница
  файл/указание меняется без изменения кода.
