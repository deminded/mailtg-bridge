# Функция: Доставка личных сообщений батчем на почту

## Цель
Держать пользователя в курсе личной переписки Telegram по почте: новые личные сообщения приходят
письмом на настроенный ящик, сгруппированные по собеседнику за такт сбора.

## Инициатор
Расписание (такт сбора входящих).

## Триггер
Наступление такта сбора при наличии новых личных сообщений после курсора диалога.

## Предусловия
Мост включён; сессия доступа к Telegram действительна; конфигурация валидна (целевой ящик, доступ).

## Основной поток
1. Система опрашивает Telegram и получает новые личные сообщения по каждому диалогу после курсора.
2. Система пропускает личные сообщения (личка адресована всегда).
3. Система группирует сообщения каждого диалога в батч за такт в хронологическом порядке.
4. Система формирует одно письмо на диалог: тег источника (напр. `dm:<отправитель>`), для каждого
   сообщения — отправитель, время, текст; медиа оформляется отдельной функцией (-> fn-media-in-email).
5. Система отправляет письмо на целевой ящик.
6. Система фиксирует запись связки (письмо ↔ диалог) и продвигает курсор диалога.

## Альтернативные потоки
- В диалоге нет новых сообщений за такт → письмо для этого диалога не формируется.
- В диалоге несколько новых сообщений за такт → одно письмо со списком, а не несколько писем.
- Сообщение является собственным эхом пользователя → отсеивается (анти-петля), в батч не входит.

## Исключительные потоки (ошибки, отказы)
- Отправка письма не удалась → курсор диалога НЕ продвигается, запись связки не создаётся;
  наблюдаемый исход: доставка отложена, на следующем такте сообщения пересобираются без дубля.
- Опрос Telegram недоступен (сеть/сессия) → такт пропускается без потери состояния; наблюдаемый
  исход: при восстановлении сбор продолжается с сохранённого курсора, потери сообщений нет.
- Мост выключен → сбор и доставка не выполняются (наблюдаемый исход: писем нет до включения).

## Результат
Одно письмо-батч на диалог на целевом ящике; запись связки; продвинутый курсор диалога.

## Постусловия
Доставленные сообщения помечены доставленными; повторный опрос/перезапуск не создаёт дублей.

## Критерии приёмки
- AC-01: дано новое личное сообщение, когда наступает такт сбора, тогда в пределах интервала
  поллинга на целевой ящик приходит письмо с корректными отправителем, текстом, временем и тегом
  источника.
- AC-02: дано диалог с N>1 новыми сообщениями за такт, когда наступает такт, тогда приходит ровно
  одно письмо со списком из N сообщений в хронологическом порядке.
- AC-03: дано уже доставленное сообщение, когда происходит перезапуск или повторный опрос, тогда
  сообщение не доставляется повторно (0 дублей).
- AC-04: дано неуспешная отправка письма, когда такт завершается, тогда курсор не продвинут и на
  следующем такте сообщения доставляются без дубля.
