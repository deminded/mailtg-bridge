# НФТ: приватность (данные в логах)

## Производительность
- Не применимо.

## SLA / SLO
- Не применимо.

## Безопасность
- NFR-PRIV-02: секреты (учётные данные почты, секрет доступа к Telegram) не появляются в логах.
  Индикатор — число вхождений секретов в логах = 0; популяция — все журналы; точка съёма —
  инспекция/скан логов.

## Отказоустойчивость
- Не применимо.

## Трассируемость и наблюдаемость
- NFR-PRIV-03: журналы содержат метаданные обработки (диалог, тег источника, счётчики), достаточные
  для диагностики, но не содержание сообщений. Индикатор — диагностируемость такта по метаданным без
  тел (да/нет); точка съёма — обзор формата логов.

## Ограничения по данным
- NFR-PRIV-01: тела сообщений не записываются в логи. Индикатор — число фрагментов тел сообщений в
  логах = 0; популяция — все журналы; точка съёма — инспекция/греп логов.

## Связи
- Паспорт: -> as-mailtg-bridge
- Функции: -> fn-dm-batch-to-email, -> fn-email-reply-to-tg
