- Профиль формальности: **S (Standard)**. Явная проверка триггеров C:

  - Персональные данные? — содержание личной переписки проходит через мост, НО субъект данных =
    сам оператор (свои сообщения в свой ящик); стороннего процессинга PII в масштабе нет. Приватность
    закрыта -> nfr-privacy (нет тел/секретов в логах).

    -> rules-security / -> nfr-security (предупреждение в README).
  - Необратимая миграция? — нет.
  Итог: не C. При этом blast-radius сессии (полный доступ к аккаунту при краже) поднимает C-стиль
  контролей точечно: -> nfr-security (600, secret-scan, вне репо), -> nfr-privacy, -> rules-security.
- found: AS-IS/TO-BE (у TKT с историей), критерии приёмки (наблюдаемые), границы скоупа, поля
  «Изменение БД/API/интеграции», релиз/версия — присутствуют по всем тикетам. Арх-скетч: граница,
  соседи и направления, точки хранения состояния — присутствуют.
- missing-blocking: нет.
- missing-non-blocking: TKT-4/5/7/8 не содержат явных секций AS-IS (помечены «—») — норма для
  новых требований, не блок; поведение при частичном отказе добрано из решений владельца и
  зафиксировано в исключительных потоках функций.
