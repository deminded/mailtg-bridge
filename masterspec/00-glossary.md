# Глоссарий: mailtg-bridge

| Термин | Определение | Не путать с |
|---|---|---|
| Анти-петля | Правило, не позволяющее собственному эху пользователя (сообщению, опубликованному мостом) снова уйти письмом | Дедуп (защита от повторной доставки того же сообщения) |
| Батч диалога | Упорядоченная группа адресованных сообщений одного диалога за один такт сбора; единица доставки (одно письмо на диалог за такт) | Отдельное сообщение; тред письма |
| Белый список источников | Перечень каналов/групп, обновления которых доставляются без упоминания | Список доверенных адресов команд |
| Bridged-письмо (мостовое письмо) | Письмо, сформированное из батча и отправленное с ящика моста B на адрес пользователя U; несёт тег источника и запись связки | Обычное письмо; подтверждение команды |
| Гейт / гейтинг адресованности | Фильтр «доставлять/не доставлять»: личка — всегда; каналы/группы — по белому списку или упоминанию | Контентная фильтрация (по смыслу текста — не входит) |
| Гибридная авторизация | Интерактивный setup порождает сессию один раз; фоновый сервис потребляет её неинтерактивно | Полностью интерактивный запуск; предзаготовленный session-файл |
| Дедуп | Защита от повторной доставки уже отправленного сообщения (курсор диалога + журнал связки), идемпотентно к перезапуску | Анти-петля |
| Диалог / Источник | Устойчивый контейнер переписки: личный собеседник, канал, группа или топик; единица батчинга и ключ маршрутизации ответа | Отдельное сообщение |
| Доверенный адрес | Адрес пользователя (U) — единственный, с которого мост принимает ответы и команды (пришедшие на ящик моста B, in-reply-to доставленного) | Ящик моста B (аккаунт-отправитель) |
| Журнал связки (ledger) | Связь bridged-письма с диалогом Telegram для маршрутизации ответа и дедупа; ключуется по диалогу | Курсор диалога (отметка прочитанного) |
| Инлайн-изображение | Изображение из Telegram, встроенное в HTML-тело письма, а не приложенное файлом | Приложенный файл; текстовое указание на вложение |
| Курсор диалога (last-id) | Покурсорная отметка «докуда прочитан диалог»; продвигается только после успешной отправки письма | Журнал связки |
| Мост выключен/включён (состояние моста) | Единый переключатель доставки; при «выключено» приём и публикация приостановлены; переживает перезапуск | Недоступность сессии/сети (временный сбой, не состояние) |
| Пользовательский клиент (userbot) | Доступ к Telegram от лица пользователя (видит личку, публикует «от себя»); несёт риск по правилам сервиса | Бот (не видит личку, пишет «от бота») |
| Порог вложений | Настраиваемый размер, разделяющий приложенные файлом вложения и вложения-указания | Лимит размера письма |
| Провайдер-нейтральность | Работа с любым почтовым сервером по адресу/порту/кредам из конфигурации, без правок кода | Мультиящик (несколько целевых ящиков) |
| Такт сбора / такт отправки | Период опроса (поллинг) входящих/исходящих; периодичности задаются конфигурацией независимо | Демон (непрерывный режим — вне скоупа версии 1) |
| Тег источника | Метка диалога в письме (напр. `dm:<отправитель>`), однозначно идентифицирующая диалог для маршрутизации ответа | Тема письма |
| Упоминание | Прямое обращение к пользователю в канале/группе, основание для доставки из невайтлист-источника | Белый список источников |
| Ящик моста (B) | Выделенный почтовый аккаунт под управлением моста: отправляет доставки (SMTP) и опрашивается на ответы/команды (IMAP) | Адрес пользователя (U); «целевой ящик» как смешанный термин |
| Адрес пользователя (U) | Личный почтовый адрес пользователя: адрес назначения доставки (целевой ящик доставки) и единственный доверенный отправитель ответов/команд | Ящик моста (B) |
| Глубокая ссылка (deep-link) | Ссылка в письме на оригинал в Telegram: `t.me/<user\|channel>/<msg>` для публичных (username), `t.me/c/<internal>/<msg>` для приватных групп/каналов участнику; для лички 1:1 шаримой ссылки НЕТ (best-effort `tg://` / без ссылки) | Тег источника (метка диалога, не ссылка) |
| Цитата-ответ (quoted-reply) | Показанное в письме сообщение, на которое отвечает текущее (на что и кому отвечено), как реплай в клиенте | Ответ письмом (reply в почте) |
