# Компонент: Хранилище состояния (курсоры, журнал связки, состояния, конфигурация)

## Назначение
Персистентная память моста, переживающая перезапуск: покурсорные отметки прочитанного на диалог,
журнал связки (письмо ↔ диалог), единый переключатель доставки, здоровье сессии, множество потреблённых
писем и доступ к конфигурации. Держит инварианты целостности (курсор монотонен и продвигается только
после успешной отправки; журнал ограничен политикой хранения), обеспечивающие at-least-once и дедуп.

## Ответственность
- Чтение конфигурации (доступ Telegram + путь к сессии, креды B, адрес U, белый список, политика
  упоминаний, порог вложений, лимит размера письма, retention, интервалы сбора/отправки, секрет-токен).
- Ведение курсора диалога: чтение и монотонное продвижение только по факту успешной отправки письма.
- Ведение журнала связки: добавление записи (Message-ID ↔ диалог), поиск по Message-ID, вычистка по retention.
- Ведение состояния моста (включён/выключен) — singleton, переживает перезапуск.
- Ведение состояния сессии (действительна/недействительна) — блокирующее, отдельно от переключателя.
- Ведение множества потреблённых писем (ответы/команды) — ограниченное, для идемпотентности приёма.

## Не делает
- Не производит сетевых операций (Telegram/почта) — только хранит и отдаёт состояние.
- Не решает бизнес-логику (гейт, деградация) — предоставляет данные владельцам возможностей.
- Не шифрует хранилище (вне скоупа версии 1 — -> rules-security): защита правами доступа и размещением
  вне репозитория.

## Возможности (capabilities)

### cap-read-config
- **Что делает:** отдаёт валидированную конфигурацию потребителям; при невалидной — явная ошибка старта.
- **Вход (логически):** источник конфигурации (файл вне репозитория).
- **Выход (логически):** параметры развёртывания (см. Ответственность).
- **Используется функциями АС/ФП:** -> fn-first-run-setup, -> fn-dm-batch-to-email, -> fn-channel-update-to-email

### cap-manage-cursor
- **Что делает:** читает курсор диалога и продвигает его строго после успешной отправки письма; при
  сбое отправки/записи связки НЕ продвигает (пересбор на следующем такте, дубль приемлем, потеря — нет).
- **Вход (логически):** идентификатор диалога; новый курсор; признак успеха отправки+связки.
- **Выход (логически):** сохранённый (монотонный) курсор.
- **Используется функциями АС/ФП:** -> fn-dm-batch-to-email, -> fn-channel-update-to-email

### cap-manage-ledger
- **Что делает:** добавляет запись связки (письмо ↔ диалог), ищет запись по Message-ID письма-родителя,
  вычищает устаревшие записи по политике хранения (журнал не растёт неограниченно).
- **Вход (логически):** Message-ID, диалог, тип источника, отправитель; запрос поиска; политика retention.
- **Выход (логически):** созданная/найденная запись; ограниченный по объёму журнал.
- **Используется функциями АС/ФП:** -> fn-dm-batch-to-email, -> fn-email-reply-to-tg

### cap-manage-bridge-state
- **Что делает:** читает и переключает единый признак «доставка включена/выключена»; идемпотентно к
  повтору команды; значение переживает перезапуск.
- **Вход (логически):** запрос чтения / команда «включить»/«выключить».
- **Выход (логически):** текущее состояние моста.
- **Используется функциями АС/ФП:** -> fn-bridge-control-by-email

### cap-manage-session-health
- **Что делает:** читает и устанавливает признак «сессия действительна/недействительна»; недействительность
  — блокирующее состояние до повторной инициализации; повторного уведомления в этом состоянии нет.
- **Вход (логически):** запрос чтения / установка «недействительна»/«действительна».
- **Выход (логически):** текущее здоровье сессии.
- **Используется функциями АС/ФП:** -> fn-first-run-setup

### cap-manage-consume-markers
- **Что делает:** хранит идентификаторы уже потреблённых писем (ответы/команды) для однократной обработки;
  множество ограничено (в согласии с retention журнала).
- **Вход (логически):** идентификатор письма; запрос «потреблено ли».
- **Выход (логически):** признак «потреблено»; обновлённое множество.
- **Используется функциями АС/ФП:** -> fn-email-reply-to-tg, -> fn-bridge-control-by-email

## Зависимости

  (-> nfr-security, права 600).
