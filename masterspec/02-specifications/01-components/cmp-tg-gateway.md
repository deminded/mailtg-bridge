# Компонент: Шлюз Telegram (пользовательская сессия)

## Назначение
Единственный владелец пользовательской сессии Telegram (MTProto): читает входящие диалоги/сообщения
после курсора, решает адресованность (гейт), отсеивает собственное эхо, скачивает медиа и публикует
ответ «от лица пользователя». Оба направления идут через одну сессию, поэтому объединены в один шлюз
(FloodWait и здоровье сессии — общие для чтения и записи; см. route-run-spec «Коллапсы»).

## Ответственность
- Итерация отслеживаемых диалогов и выборка новых сообщений строго после курсора диалога.
- Применение гейта адресованности к каждому сообщению до батчинга (личка — всегда; канал/группа — по
  белому списку ИЛИ упоминанию согласно политике).
- Распознавание собственного эха моста (анти-петля) — исключение из выдачи.
- Скачивание вложений сообщения (изображения и файлы) по запросу формирования письма.
- Публикация текста ответа в целевой диалог от лица пользователя.
- Классификация ошибок сессии: FloodWait / преходящая сетевая / сессия отозвана-недействительна.

## Не делает
- Не формирует письма и не рендерит медиа (это -> cmp-email-out) — отдаёт сырые данные сообщения/вложений.
- Не хранит курсор/журнал/состояние (это -> cmp-state-store) — получает курсор на вход.
- Не решает, когда запускать такт и когда отступать (это -> cmp-bridge-orchestrator) — сообщает класс ошибки.
- Не отвечает на конкретное сообщение внутри диалога (reply-to сообщения — задел версии 2): публикует в диалог.

## Возможности (capabilities)

### cap-fetch-since-cursor
- **Что делает:** для каждого отслеживаемого диалога возвращает новые сообщения после курсора в
  хронологическом порядке с метаданными (автор, @username, время, форматирование, цитата-ответ,
  признак упоминания, перечень вложений).
- **Вход (логически):** список диалогов и их курсоры.
- **Выход (логически):** упорядоченные сообщения по диалогу либо пусто.
- **Используется функциями АС/ФП:** -> fn-dm-batch-to-email, -> fn-channel-update-to-email

### cap-apply-addressing-gate
- **Что делает:** пропускает/отсеивает сообщение канала/группы по белому списку ИЛИ упоминанию согласно
  политике упоминаний; личку пропускает всегда.
- **Вход (логически):** сообщение с типом источника и признаком упоминания; белый список; политика упоминаний.
- **Выход (логически):** решение «пропустить» / «отсеять» (без промежуточных состояний).
- **Используется функциями АС/ФП:** -> fn-channel-update-to-email, -> fn-dm-batch-to-email

### cap-detect-own-echo
- **Что делает:** распознаёт сообщение, опубликованное самим мостом от лица пользователя, и исключает
  его из выдачи (анти-петля).
- **Вход (логически):** сообщение (признак исходящего/авторства пользователя + метка опубликованных мостом).
- **Выход (логически):** признак «собственное эхо» → отсев.
- **Используется функциями АС/ФП:** -> fn-dm-batch-to-email

### cap-download-media
- **Что делает:** скачивает содержимое вложения сообщения (байты + имя/размер/тип) для последующего
  представления в письме; при неудаче возвращает признак недоступности.
- **Вход (логически):** ссылка на вложение сообщения.
- **Выход (логически):** содержимое и метаданные вложения либо признак «недоступно».
- **Используется функциями АС/ФП:** -> fn-media-in-email

### cap-post-as-user
- **Что делает:** публикует текст ответа в целевой диалог от лица пользователя; порождённое сообщение
  помечается для анти-петли.
- **Вход (логически):** идентификатор диалога, текст ответа.
- **Выход (логически):** факт публикации (успех/неуспех), идентификатор опубликованного сообщения.
- **Используется функциями АС/ФП:** -> fn-email-reply-to-tg

### cap-surface-session-errors
- **Что делает:** приводит ошибки MTProto к трём различимым классам для оркестратора — FloodWait
  (отступить), преходящая (повторить позже), сессия недействительна/отозвана (блокирующее).
- **Вход (логически):** ошибка вызова Telegram.
- **Выход (логически):** класс ошибки.
- **Используется функциями АС/ФП:** -> fn-first-run-setup, -> fn-dm-batch-to-email

## Зависимости
- Личный аккаунт Telegram пользователя (внешняя система) — источник входящих и адресат исходящих.
- -> cmp-state-store — получить путь к сессии и белый список/политику (через конфигурацию), курсоры.
