## Назначение
Хранит состояние, переживающее перезапуск и обеспечивающее дедуп, at-least-once, маршрутизацию ответа,
анти-петлю, управление и здоровье сессии. Опора функций -> fn-dm-batch-to-email, -> fn-channel-update-to-email,
-> fn-email-reply-to-tg, -> fn-bridge-control-by-email, -> fn-first-run-setup.

## Сущности, ключевые атрибуты, связи между сущностями
Вынесены в машинный сайдкар `data-bridge-store.schema.json` (JSON Schema 2020-12, `$defs`):
`DialogCursor` (Курсор диалога), `LedgerRecord` (Запись связки/ledger), `ConsumeMarker` (Маркер
потребления), `EchoMarker` (Маркер эха), `BridgeState` (Состояние моста, singleton), `SessionState`
(Состояние сессии, singleton), `Configuration` (только чтение). Назначение каждой сущности и связи

## Состояния и переходы

### Состояние моста (хранимое, singleton)
| Состояние | Событие | → Результат (или ЗАПРЕЩЕНО) | Момент |
|---|---|---|---|
| включён | команда «выключить» (доверенная) | выключен + подтверждение | при записи |
| включён | команда «включить» (уже включён) | включён (идемпотентно) | при записи |
| включён | команда недоверенная/без токена | включён (игнор + лог) | при записи |
| включён | наступил такт | включён (доставка/публикация идут) | по расписанию |
| включён | перезапуск | включён (сохранено) | при чтении |
| выключен | команда «включить» (доверенная) | включён + подтверждение | при записи |
| выключен | команда «выключить» (уже выключен) | выключен (идемпотентно) | при записи |
| выключен | команда недоверенная/без токена | выключен (игнор + лог) | при записи |
| выключен | наступил такт | выключен (доставка/публикация приостановлены; команды исполняются) | по расписанию |
| выключен | перезапуск | выключен (сохранено) | при чтении |

### Состояние сессии (хранимое, singleton; недействительность — блокирующее)
| Состояние | Событие | → Результат (или ЗАПРЕЩЕНО) | Момент |
|---|---|---|---|
| действительна | опрос/публикация проходят | действительна | по расписанию |
| действительна | Telegram отклонил сессию при ЛЮБОЙ операции (опрос/скачивание/публикация, вкл. частичный отказ) | недействительна (опрос стоп; notified←false→уведомить U) | при записи |
| действительна | перезапуск | действительна (сохранено) | при чтении |
| недействительна | наступил такт | недействительна (опрос НЕ идёт; при notified=true повтора уведомления нет) | по расписанию |
| недействительна | уведомление отправлено | недействительна, notified=true | при записи |
| недействительна | повторная интерактивная инициализация | действительна, notified=false | при записи |
| недействительна | перезапуск | недействительна (блокирующее сохранено) | при чтении |

### Курсор диалога (хранимое значение, монотонное — не автомат)
| Состояние | Событие | → Результат (или ЗАПРЕЩЕНО) | Момент |
|---|---|---|---|
| last_id=k | письмо батча отправлено И связка создана | last_id←макс. выбранный в такте msg_id (high-watermark; k'≥k) | при записи |
| last_id=k | отправка/запись связки не удалась | last_id=k (НЕ продвигать; пересбор) | при записи |
| last_id=k | попытка отката назад | ЗАПРЕЩЕНО (монотонность: курсор не убывает) | — |

Курсор — монотонный high-watermark: удалённые/отсутствующие id (сообщение удалено до обработки, дыра в
нумерации) перешагиваются и НЕ стопорят продвижение за них последующих сообщений (-> alg-batch-per-dialog-cycle).

Жизненный цикл сообщения (fetched→gated→batched→delivered→replied) — концептуальный, в -> cdm-bridge
(сообщения не хранятся мостом, хранится только курсор).

## Ограничения
- message_id записи связки — уникален; mail_ref маркера потребления — уникален; dialog_id курсора — уникален.
- last_id монотонно не убывает; продвигается до макс. выбранного в такте msg_id (high-watermark) и НЕ
  опережает ни одно прошедшее гейт сообщение, не вошедшее в успешно доставленный батч (удалённые/отсеянные
  id перешагиваются, -> alg-batch-per-dialog-cycle).
- Журнал связки, маркеры потребления и эха ОГРАНИЧЕНЫ политикой retention: вычистка — oldest-first по
  возрасту ИЛИ количеству, вычищаются только РАЗРЕШЁННЫЕ записи (доставленная связка / потреблённое письмо /
  эхо-маркер за пределами анти-петля-окна); конкретные значения возраста/лимита — конфигурация. Не растут
  неограниченно (-> nfr-operability NFR-OPS-07); согласованность вычистки — см. -> alg-dedup-idempotency.
- Состояние моста и состояние сессии — ровно по одному экземпляру.
- Файлы состояния и сессии — вне дерева репозитория, права только для владельца (600); не шифруются (вне
  скоупа версии 1, принятый остаточный риск -> rules-security). Выбор «без шифрования» СДЕЛАН (-> nfr-security NFR-SEC-05).
- Конфигурация только читается сервисом; смена провайдера/аккаунта — правкой конфигурации (-> nfr-deployability).
