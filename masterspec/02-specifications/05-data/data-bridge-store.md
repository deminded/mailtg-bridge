## Назначение
Хранит состояние, переживающее перезапуск и обеспечивающее дедуп, at-least-once, маршрутизацию ответа,
анти-петлю, управление и здоровье сессии. Опора функций -> fn-dm-batch-to-email, -> fn-channel-update-to-email,
-> fn-email-reply-to-tg, -> fn-bridge-control-by-email, -> fn-first-run-setup.

## Сущности
- Курсор диалога — отметка «докуда прочитан диалог»; ключ дедупа доставки.
- Запись связки (ledger) — связь мостового письма с диалогом; ключ маршрутизации ответа.
- Маркер потребления — идентификаторы обработанных писем-ответов/команд (идемпотентность приёма).
- Маркер эха — идентификаторы сообщений, опубликованных мостом (анти-петля).
- Состояние моста — singleton-переключатель доставки (включён/выключен).
- Состояние сессии — singleton-здоровье сессии (действительна/недействительна).
- Конфигурация — параметры развёртывания (только чтение; источник — файл вне репозитория).

## Ключевые атрибуты
**Курсор диалога:** dialog_id (идентификатор, ключ); source_type (перечисление dm/channel/group/topic,
-> dict-source-type); last_id (число, монотонное); whitelisted (флаг); source_tag (строка); updated_at (дата-время).

**Запись связки (ledger):** message_id (идентификатор письма, ключ, уникален); dialog_id (идентификатор,
внешняя ссылка на Курсор диалога); source_type (перечисление); sender (строка — отправитель исходного
сообщения/тег); delivered_at (дата-время).

**Маркер потребления:** mail_ref (идентификатор письма: message_id/UID, ключ, уникален); consumed_at (дата-время).

**Маркер эха:** posted_msg_id (идентификатор сообщения в диалоге, ключ); dialog_id (идентификатор);
posted_at (дата-время).

**Состояние моста:** enabled (флаг); updated_at (дата-время). Экземпляр один (singleton).

**Состояние сессии:** valid (флаг); notified (флаг «уведомление о недействительности отправлено»);
updated_at (дата-время). Экземпляр один (singleton).

**Конфигурация (только чтение):** tg_access (доступ Telegram), session_path (строка-путь); B_imap {host,
port, credentials}, B_smtp {host, port, credentials}; U_address (строка-адрес); whitelist (набор источников);
mention_policy (перечисление: все / выбранные / форсирование по списку) + mention_list (набор); attachment_threshold
(размер); email_size_limit (размер); retention (длительность/лимит); collect_interval (длительность);
send_interval (длительность); command_token (строка, опционально). Секреты — вне репозитория, не в логах
(-> rules-security, -> nfr-privacy).

## Связи между сущностями
- Курсор диалога 1 — N Запись связки (много писем на один диалог; ключ маршрутизации — диалог, R-06 в -> cdm-bridge).
- Запись связки N — 1 Курсор диалога (по dialog_id).
- Маркер эха N — 1 Курсор диалога (по dialog_id).
- Состояние моста, Состояние сессии — singleton (по одному экземпляру).
- Конфигурация 1 — N Курсор диалога (whitelist/теги отбирают/именуют диалоги, R-07).

## Состояния и переходы

### Состояние моста (хранимое, singleton)
| Состояние | Событие | → Результат (или ЗАПРЕЩЕНО) | Момент |
|---|---|---|---|
| включён | команда «выключить» (доверенная) | выключен + подтверждение | при записи |
| включён | команда «включить» (уже включён) | включён (идемпотентно) | при записи |
| включён | команда недоверенная/без токена | включён (игнор + лог) | при записи |
| включён | наступил такт | включён (доставка/публикация идут) | по расписанию |
| включён | перезапуск | включён (сохранено) | при чтении |
| выключен | команда «включить» (доверенная) | включён + подтверждение | при записи |
| выключен | команда «выключить» (уже выключен) | выключен (идемпотентно) | при записи |
| выключен | команда недоверенная/без токена | выключен (игнор + лог) | при записи |
| выключен | наступил такт | выключен (доставка/публикация приостановлены; команды исполняются) | по расписанию |
| выключен | перезапуск | выключен (сохранено) | при чтении |

### Состояние сессии (хранимое, singleton; недействительность — блокирующее)
| Состояние | Событие | → Результат (или ЗАПРЕЩЕНО) | Момент |
|---|---|---|---|
| действительна | опрос/публикация проходят | действительна | по расписанию |
| действительна | Telegram отклонил сессию | недействительна (опрос стоп; notified←false→уведомить U) | при записи |
| действительна | перезапуск | действительна (сохранено) | при чтении |
| недействительна | наступил такт | недействительна (опрос НЕ идёт; при notified=true повтора уведомления нет) | по расписанию |
| недействительна | уведомление отправлено | недействительна, notified=true | при записи |
| недействительна | повторная интерактивная инициализация | действительна, notified=false | при записи |
| недействительна | перезапуск | недействительна (блокирующее сохранено) | при чтении |

### Курсор диалога (хранимое значение, монотонное — не автомат)
| Состояние | Событие | → Результат (или ЗАПРЕЩЕНО) | Момент |
|---|---|---|---|
| last_id=k | письмо батча отправлено И связка создана | last_id←последний msg_id батча (k'≥k) | при записи |
| last_id=k | отправка/запись связки не удалась | last_id=k (НЕ продвигать; пересбор) | при записи |
| last_id=k | попытка отката назад | ЗАПРЕЩЕНО (монотонность: курсор не убывает) | — |

Жизненный цикл сообщения (fetched→gated→batched→delivered→replied) — концептуальный, в -> cdm-bridge
(сообщения не хранятся мостом, хранится только курсор).

## Ограничения
- message_id записи связки — уникален; mail_ref маркера потребления — уникален; dialog_id курсора — уникален.
- last_id монотонно не убывает и не опережает фактически доставленное.
- Журнал связки, маркеры потребления и эха ОГРАНИЧЕНЫ политикой retention (устаревшее вычищается) —
  не растут неограниченно (-> nfr-operability NFR-OPS-07); согласованность вычистки — см. -> alg-dedup-idempotency.
- Состояние моста и состояние сессии — ровно по одному экземпляру.
- Файлы состояния и сессии — вне дерева репозитория, права только для владельца (600); не шифруются (вне
  скоупа версии 1, принятый остаточный риск -> rules-security). Выбор «без шифрования» СДЕЛАН (-> nfr-security NFR-SEC-05).
- Конфигурация только читается сервисом; смена провайдера/аккаунта — правкой конфигурации (-> nfr-deployability).
