## Назначение
Полный перечень настроек развёртывания моста: имя ключа, тип, дефолт, смысл, секретность. Читается
один раз при старте процесса (`Settings.from_env`), далее неизменна на время работы — это и есть
«Конфигурация (только чтение)» из -> data-bridge-store. Опора -> fn-first-run-setup: смена почтового
провайдера/аккаунта или whitelist — правкой конфигурации, без изменения кода (-> nfr-deployability
NFR-DEPLOY-02). Секреты (столбец «Секрет: да») не вносятся в код/репозиторий и не появляются в логах
(-> rules-security, -> nfr-security, -> nfr-privacy).

## Ключи конфигурации
Полный перечень (обязательные без дефолта + параметры с дефолтом: почта B, маршрутизация/отбор
источников Telegram, такты/лимиты/деградация, хранение и вычистка, управление) вынесен в машинный
сайдкар `data-config.schema.json` (JSON Schema 2020-12; плоская config-схема — свойства прямо в
`properties`, -> migration-rules.md §2). Тип/дефолт/секретность каждого ключа — в самом сайдкаре
(`type`/`default`/`description` с пометкой «секрет —» у секретных ключей); группировка по темам
(почта B — соединение / маршрутизация / такты / retention / управление) сохранена как порядок ключей
и в `description`. Обязательные-без-дефолта ключи — в `required` сайдкара; их отсутствие/пустота
завершает старт ошибкой конфигурации (как в исходной прозе).

## Валидация и связи между параметрами
- `ATTACHMENT_THRESHOLD_BYTES` ДОЛЖЕН быть меньше `EMAIL_SIZE_LIMIT_BYTES` (иначе ошибка конфигурации
  при старте).
- `ECHO_RETENTION_SECONDS` НЕ ДОЛЖЕН превышать `RETENTION_SECONDS`.
- `BACKOFF_MIN_SECONDS` НЕ ДОЛЖЕН превышать `BACKOFF_MAX_SECONDS`.
- Пути (`TG_SESSION_PATH`, `STATE_DB_PATH`, `LOCK_PATH`, `TEMP_DIR`) ДОЛЖНЫ быть абсолютными.
- `B_ADDRESS`/`U_ADDRESS` ДОЛЖНЫ быть валидным простым email-адресом (без имени/уголков).
- `*_JSON`-ключи (`WHITELIST_JSON`, `MENTION_LIST_JSON`, `BOT_LIST_JSON`) — JSON-массив строк-источников
  в формате `canonical_source`: `@username` (≥3 символов, регистронезависимо) либо числовой id (со
  знаком), опционально с суффиксом `:topic_id`.
- Все целочисленные параметры (кроме дефолта `TG_API_ID`, где отсутствие ловится обязательностью)
  ДОЛЖНЫ быть положительными; нечисловое или неположительное значение — ошибка конфигурации при старте.
- Задаётся либо через переменные окружения процесса, либо через файл (`KEY=value` построчно,
  опциональные кавычки), переданный отдельно; окружение процесса имеет приоритет над файлом.
