# Схема данных: контракт конфигурации

<!-- Контракт конфигурации (env / .env-файл), единый источник истины — `Settings.from_env` в
     src/mailtg_bridge/config.py. Все env-ключи, упоминаемые голым именем в tc-acc/lp/nfr, трассируются
     сюда — контракт определяется здесь один раз, а не дублируется по слоям (one-layer). -->

## Назначение
Полный перечень настроек развёртывания моста: имя ключа, тип, дефолт, смысл, секретность. Читается
один раз при старте процесса (`Settings.from_env`), далее неизменна на время работы — это и есть
«Конфигурация (только чтение)» из -> data-bridge-store. Опора -> fn-first-run-setup: смена почтового
провайдера/аккаунта или whitelist — правкой конфигурации, без изменения кода (-> nfr-deployability
NFR-DEPLOY-02). Секреты (столбец «Секрет: да») не вносятся в код/репозиторий и не появляются в логах
(-> rules-security, -> nfr-security, -> nfr-privacy).

## Обязательные параметры
Без значения по умолчанию — старт завершается ошибкой конфигурации, если ключ отсутствует или пуст.

| Ключ | Тип | Дефолт | Смысл | Секрет |
|---|---|---|---|---|
| `TG_API_ID` | целое, >0 | — (обязателен) | ID приложения Telegram (my.telegram.org) | нет |
| `TG_API_HASH` | строка | — (обязателен) | Hash приложения Telegram (my.telegram.org) | да |
| `TG_SESSION_PATH` | путь (абсолютный) | — (обязателен) | Файл сессии Telethon; компрометация = полный доступ к аккаунту Telegram | да |
| `B_ADDRESS` | email | — (обязателен) | Адрес почтового ящика моста B (выделенный аккаунт) | нет |
| `B_USERNAME` | строка | — (обязателен) | Логин IMAP/SMTP ящика B | да |
| `B_PASSWORD` | строка | — (обязателен) | Пароль/app-пароль ящика B | да |
| `B_IMAP_HOST` | строка | — (обязателен) | Хост IMAP-сервера ящика B | нет |
| `B_SMTP_HOST` | строка | — (обязателен) | Хост SMTP-сервера ящика B | нет |
| `U_ADDRESS` | email | — (обязателен) | Адрес пользователя U — получатель писем моста и единственный доверенный отправитель команд/ответов | нет |
| `STATE_DB_PATH` | путь (абсолютный) | — (обязателен) | Файл персистентного состояния (курсоры/ledger/маркеры), -> data-bridge-store; вне репозитория, права 600 | нет |
| `LOCK_PATH` | путь (абсолютный) | — (обязателен) | Lock-файл единственного экземпляра процесса | нет |
| `TEMP_DIR` | путь (абсолютный) | — (обязателен) | Каталог временных файлов вложений (очищается после каждого такта) | нет |

## Параметры с дефолтом

### Почта B — соединение
| Ключ | Тип | Дефолт | Смысл | Секрет |
|---|---|---|---|---|
| `B_IMAP_PORT` | целое, >0 | `993` | Порт IMAP ящика B | нет |
| `B_SMTP_PORT` | целое, >0 | `465` | Порт SMTP ящика B | нет |
| `B_IMAP_SECURITY` | перечисление `ssl \| starttls` | `ssl` | Режим шифрования IMAP-соединения | нет |
| `B_SMTP_SECURITY` | перечисление `ssl \| starttls` | `ssl` | Режим шифрования SMTP-соединения | нет |
| `SAVE_SENT_COPY` | булево (`1/true/yes`) | `true` | Сохранять ли копию исходящего ответа в папку отправленных ящика B | нет |
| `SENT_FOLDER` | строка | `Sent` | Имя IMAP-папки для копии отправленных | нет |

### Маршрутизация и отбор источников Telegram
| Ключ | Тип | Дефолт | Смысл | Секрет |
|---|---|---|---|---|
| `WHITELIST_JSON` | JSON-массив источников | `[]` | Белый список каналов/групп/пользователей, отслеживаемых всегда (canonical_source: `@username` ≥3 симв. либо числовой id, опционально `:topic_id`) | нет |
| `MENTION_POLICY` | перечисление `all \| selected \| forced_list` | `selected` | Политика отслеживания источников вне whitelist по признаку упоминания пользователя | нет |
| `MENTION_LIST_JSON` | JSON-массив источников | `[]` | Список источников для режима `forced_list` | нет |
| `DISCOVER_ALL_DIALOGS` | булево (`1/true/yes`) | `false` | Отслеживать все диалоги аккаунта, а не только whitelist ∪ mention_list | нет |
| `BOT_POLICY` | перечисление `none \| all \| selected` | `all` | Политика доставки сообщений, авторство которых — бот (`is_bot=true`) | нет |
| `BOT_LIST_JSON` | JSON-массив источников | `[]` | Список ботов (`@username`), разрешённых при `BOT_POLICY=selected` | нет |

### Такты, лимиты, деградация
| Ключ | Тип | Дефолт | Смысл | Секрет |
|---|---|---|---|---|
| `COLLECT_INTERVAL_SECONDS` | целое, >0 | `60` | Период такта сбора Telegram (личка + каналы/группы) | нет |
| `SEND_INTERVAL_SECONDS` | целое, >0 | `30` | Период такта опроса ящика B (ответы пользователя + команды управления) | нет |
| `TG_FETCH_LIMIT` | целое, >0 | `100` | Макс. число сообщений на диалог за один такт сбора | нет |
| `ATTACHMENT_THRESHOLD_BYTES` | целое, >0 | `10485760` (~10 МБ) | Порог размера файла: ниже — встраивается/прикладывается, выше — текстовое указание вместо содержимого | нет |
| `EMAIL_SIZE_LIMIT_BYTES` | целое, >0 | `25165824` (~24 МБ) | Лимит совокупного размера письма-батча (MIME) | нет |
| `BACKOFF_MIN_SECONDS` | целое, >0 | `30` | Минимальная пауза экспоненциального backoff при повторных ошибках (FloodWait/сетевые) | нет |
| `BACKOFF_MAX_SECONDS` | целое, >0 | `3600` | Максимальная пауза backoff | нет |

### Хранение и вычистка (retention)
| Ключ | Тип | Дефолт | Смысл | Секрет |
|---|---|---|---|---|
| `RETENTION_SECONDS` | целое, >0 | `7776000` (90 дней) | Возраст, после которого записи связки/потребления разрешено вычищать | нет |
| `RETENTION_MAX_LEDGER` | целое, >0 | `50000` | Макс. число хранимых записей связки (ledger) | нет |
| `RETENTION_MAX_CONSUMED` | целое, >0 | `50000` | Макс. число хранимых маркеров потребления | нет |
| `RETENTION_MAX_ECHO` | целое, >0 | `10000` | Макс. число хранимых маркеров эха (анти-петля) | нет |
| `ECHO_RETENTION_SECONDS` | целое, >0 | `604800` (7 дней) | Окно анти-петли: возраст, после которого маркер эха разрешено вычищать | нет |

### Управление и прочее
| Ключ | Тип | Дефолт | Смысл | Секрет |
|---|---|---|---|---|
| `COMMAND_TOKEN` | строка | `""` (пусто = не требуется) | Секрет-токен подтверждения команд управления мостом («MAILTG OFF/ON») по почте | да |
| `BOOTSTRAP_MODE` | перечисление `tail \| history` | `tail` | Режим первичной инициализации курсора диалога: с хвоста или с истории | нет |
| `TIMEZONE` | строка (IANA tz) | `America/Phoenix` | Таймзона локализации времени в письмах | нет |
| `LOG_LEVEL` | строка | `INFO` (приводится к верхнему регистру) | Уровень логирования сервиса | нет |

## Валидация и связи между параметрами
- `ATTACHMENT_THRESHOLD_BYTES` ДОЛЖЕН быть меньше `EMAIL_SIZE_LIMIT_BYTES` (иначе ошибка конфигурации
  при старте).
- `ECHO_RETENTION_SECONDS` НЕ ДОЛЖЕН превышать `RETENTION_SECONDS`.
- `BACKOFF_MIN_SECONDS` НЕ ДОЛЖЕН превышать `BACKOFF_MAX_SECONDS`.
- Пути (`TG_SESSION_PATH`, `STATE_DB_PATH`, `LOCK_PATH`, `TEMP_DIR`) ДОЛЖНЫ быть абсолютными.
- `B_ADDRESS`/`U_ADDRESS` ДОЛЖНЫ быть валидным простым email-адресом (без имени/уголков).
- `*_JSON`-ключи (`WHITELIST_JSON`, `MENTION_LIST_JSON`, `BOT_LIST_JSON`) — JSON-массив строк-источников
  в формате `canonical_source`: `@username` (≥3 символов, регистронезависимо) либо числовой id (со
  знаком), опционально с суффиксом `:topic_id`.
- Все целочисленные параметры (кроме дефолта `TG_API_ID`, где отсутствие ловится обязательностью)
  ДОЛЖНЫ быть положительными; нечисловое или неположительное значение — ошибка конфигурации при старте.
- Задаётся либо через переменные окружения процесса, либо через файл (`KEY=value` построчно,
  опциональные кавычки), переданный отдельно; окружение процесса имеет приоритет над файлом.
