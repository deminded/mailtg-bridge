# Алгоритм: гейт адресованности (что доставляется)

## Контекст применения
Применяется в -> cmp-tg-gateway/cap-apply-addressing-gate на шаге гейтинга сценария
-> scn-inbound-collect-cycle, к каждому сообщению до включения в батч. Реализует -> rules-gating.

## Вход
Сообщение (тип источника: dm/channel/group/topic; признак прямого упоминания пользователя); белый список
источников; политика упоминаний (все / выбранные / форсирование по списку).

## Выход
Решение «пропустить» или «отсеять» (без промежуточных состояний).

## Правила
1. Если тип источника — dm (личка) → пропустить. Иначе → перейти к правилу 2 (канал/группа/топик).
2. Если источник входит в белый список → пропустить. Иначе → перейти к правилу 3.
3. Если у сообщения есть прямое упоминание пользователя И политика упоминаний учитывает этот источник →
   пропустить. Иначе → отсеять (поведение по умолчанию для невайтлист-источника без учитываемого упоминания).
4. Учёт упоминания по политике (детализация правила 3): режим «все» → упоминание учитывается для любого
   источника; режим «выбранные»/«форсирование по списку» → упоминание учитывается только для источников
   из соответствующего списка конфигурации, для прочих → не учитывается (эквивалентно отсутствию упоминания).
5. Контентная (смысловая) фильтрация в гейт не входит — оценивается только адресованность.

## Порядок проверок
Порядок важен: сначала тип (dm — короткое замыкание на «пропустить»), затем белый список, затем упоминание
по политике. Личка приоритетна и не подчиняется списку/упоминанию; для каналов/групп «нет основания» = отсев.

## Причины выбора порядка
Приоритет лички и «не воспроизводить шум» из -> rules-gating (принципы принятия решений).

## Инварианты
- Решение гейта бинарно и детерминировано для одного входа.
- dm никогда не отсеивается по правилам списка/упоминания.
- Отсеянное сообщение не попадает в батч и не порождает письма; курсор при этом всё равно может двигаться
  (сообщение обработано — состояние «dropped»), см. -> cdm-bridge.

## Связи
- Правила: -> rules-gating
- Сценарии: -> scn-inbound-collect-cycle
- Компоненты: -> cmp-tg-gateway

## Evolve-дельта (13.07, живой тест)
**Ветка ботов (перед DM-проходом):** если `message.sender.is_bot`:
- `bot_policy=none` → отсеять;
- `bot_policy=all` → продолжить обычный гейт (для DM — пропуск);
- `bot_policy=selected` → пропустить только если @username бота ∈ `bot_list`, иначе отсеять.
Не-бот сообщения политикой ботов не затрагиваются. Сигнатура: `is_addressed(..., bot_policy, bot_list)`.
