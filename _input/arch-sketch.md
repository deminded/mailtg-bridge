# Архитектурный скетч — tg-mail-bridge (Telegram[Telethon] ↔ почта)

## Компоненты
- **tg-poller** — Telethon-юзерклиент: забирает новое из личек (все) и каналов/групп (по вайтлисту/упоминанию) с последнего id по каждому диалогу.
- **gate** — фильтр: личка → всё; каналы/группы → вайтлист ИЛИ прямое упоминание; политика из конфига (все/выбранные; форс-упоминаний да/нет/по списку).
- **email-out** — форматирует сообщение в письмо (тег источника, текст, вложения по размеру: мал.→файл, кр.→указание) и шлёт на целевой ящик по SMTP.
- **email-in** — IMAP-поллинг целевого ящика: ловит ответы на bridged-письма.
- **tg-sender** — Telethon: постит ответ в исходный TG-чат ОТ ЛИЦА пользователя, в правильный диалог (по ledger).
- **state** — файловое: per-dialog `last_id` (докуда прочитан каждый диалог) + `ledger` (email Message-ID ↔ {tg_chat, tg_msg, source, sender}) — роутинг ответа и дедуп.

## Потоки
- **Входящий (TG → почта):** tg-poller → gate (пропустить/отсеять) → email-out (тег источника + вложения) → SMTP на ящик → запись в ledger.
- **Исходящий (почта → TG):** email-in ловит reply на bridged-письмо → по Message-ID/тегу находит запись в ledger → tg-sender постит в исходный чат от лица пользователя.

## Состояние и дедуп
- per-dialog `last_id` — монотонная отметка прочитанного по каждому диалогу (переживает рестарт).
- `ledger` — связка письмо↔TG-источник: и роутинг ответа, и защита от повторной доставки.

## Развёртывание
- Поллинг (решено): два cron-такта — tg-in и email-in — ИЛИ один цикл; периодичности сбора и отправки из конфига. Демон — опциональный режим позже.
- Конфиг — `.env` (Telegram api_id/api_hash/session, IMAP/SMTP, ящик, вайтлист, политика упоминаний, пороги, периодичности); секреты и session вне репо. Поставка: docker-compose ИЛИ systemd + `.env.example` + README.

## Переиспользование (обкатанное ядро)
- **channel-reader** — паттерн Telethon-поллинга каналов с per-channel курсором и нуджем (прообраз tg-poller/email-out).
- **bridge_poller** — гейт адресованности (mention/reply), реестр источников (прообраз gate).
- **de-agent-commons / agentcommons.email** — IMAP/SMTP, тред, дедуп-ledger (прообраз email-in/email-out/state).
- Провайдер email задаётся host/port в конфиге; TG-сессия — Telethon (arete-userbot как референс механики, но session = полный доступ к аккаунту, см. TKT-7).

## Безопасность (несущее)
- Telethon session-файл = полный доступ к аккаунту: 600, вне репо, предупреждение о ToS-риске userbot в README (TKT-7).
